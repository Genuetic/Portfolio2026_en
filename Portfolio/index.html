IBL Banca — Key Learnings

In a regulated
product,
exceptions
are the product

12 months on a critical banking system, two conflicting organisations. What no brief anticipates.

01 — Habits are design constraints

The network's operational routines weren't the problem to solve. They were the design constraint. Going into the field before wireframes changed everything.

02 — The designer as mediator

The bank and the network had opposing interests. The cascading buy-in depended on the ability to keep both parties at the table without turning them against each other.

03 — Anticipate regulatory complexity

Regulatory edge cases aren't rare: they're structural. Treating them as "a later phase" is a debt you pay on the run.

Levi's PDM — The problem

But it didn't speak.

The interface didn't communicate the underlying workflow: no status indicators, no visual hierarchy between information, no signage on which fields were required to advance in the process.

Levi's PDM — First iteration

patterns

The problem wasn't aesthetic: it was structural. The system didn't know where it stood in its own workflow. The response was to build signage before building forms: sidebar with visible states, active filter chips above the grid, a permanent status legend at the bottom of the sheet.

After

Levi's PDM — Key decisions

Where the project
demanded
real choices

Three moments where the spec wasn't enough: an independent, argued design proposal shown in context was required.

01 — Status icon: fewer elements, same meaning

The client had specified an additional composite icon to signal advancement fields. I proposed reusing the existing icon and changing its colour based on the next status: same semantics, zero new elements, zero visual noise. Two comparative screenshots, approved in the first round.

02 — Active filters: visibility without duplication

A filter with ten selected values doesn't fit in a chip. Two-layer solution: chips above the grid to show which filters are active, sidebar for granular control. Clear All and Save Filters in the toolbar.

03 — Scope change on technical specs

Late in the project the client redefined the product specs data model. I responded with a fully articulated solution in the same email: separate popups for Upper Materials and Design Specifications. Approved in the first round.

Kokous — Key decisions

Constraints
as
differentiators

Three non-obvious design choices — that turned real limitations into competitive advantage.

01 — GDPR by design, not retrofit

Elastic search full-match to avoid partial data leaks, profile photos removed, digital signature on the totem, configurable retention. Compliance was built into the design before the code.

02 — Readability before aesthetics

Kokous Room ran on low-end Android tablets: aggressive contrast, large typography, zero animations. The interface was optimised for someone reading in a hurry, in a corridor, 50 cm from the screen.

03 — Modular architecture = speed of response

Neutral design system, reusable components, team trained for rapid iteration. When COVID arrived, Kokous TH was released in weeks, not months. The foundation was already there.
